Aufgabe 1.1: World Wide Web

  1. Mit welchen fachlichen Argumenten wurde das WWW-Proposal von Tim Burners Lee abgelehnt?
  2. Was sind die fachlichen Argumente, warum das WWW dennoch ein Erfolg wurde?
  3. Was wäre der Preis für die garantierte Verhinderung von "broken links"?

Meine Lösung:

  1. Die Wagheit wurde mit der Evolvierbarkeit verwechselt. Die Hypertext Community war der Meinung, bereits weiter in der Wissenschaft gewesen zu sein, dass WWW-Proposal würde demnach den Fortschritt zurück schmeissen, vor allem durch die möglichen "broken links".

    • Die Technologien können sich unabhängig voneinander weiterentwickeln.
    • Offene Standards
    • Einfache Benutzeroberfläche

    • hohe kosten
    • überwachung und Datenschutz

Aufgabe 1.2: HTTP

  1. Sie bekommen im Browser den HTTP Status Code 200. Was bedeutet das?
  2. Sie bekommen im Browser den HTTP Status Code 301. Was hat das zu bedeuten?
  3. Sie bekommen im Browser den HTTP Status Code 400. Was hat das zu bedeuten? Was können Sie dagegen tun?
  4. Sie bekommen im Browser den HTTP Status Code 403. Was hat das zu bedeuten? Was können Sie dagegen tun?
  5. In einer Webanwendung benötigen Sie eine OPTIONS-Anfrage, die die Optionen des Servers vor dem eigentlichen Zugriff erfragen soll. Aus Performancegründen soll die Abfrage aber cacheable sein. Wie könnten Sie dafür eine Lösung angehen?

Meine Lösung:

HTTP STATUS CODE Bedeutung Optionen
200 OK (Aufruf/Anfrage wurde erfolgreich ausgeführt)
301 "Moved Permanently" (ehemalige URL wird zu einer neuen URL weiter geleitet, wenn zB URL nicht mehr existiert oder geändert wurde)
400 "Bad Request" (Anfrage wurde fehlerhaft aufgebaut, Verbindung mit Webserver wurde hergestellt, aber Webseite aufgrund eines Problems mit der Adresse nicht gefunden). zB. Überprüfen der übermittelten URL, Browser-Cache löschen, Browser-Cookies löschen, Prüfen, ob Dateiupload Serverlimit überschreitet, DNS-Cache löschen, Browser Erweiterungen deaktivieren.
403 Browser hat keine Berechtigung auf die Website zuzugreifen. Erweiterungen deaktivieren, Cache löschen Firewall deaktivieren

5. Man kann das HTTP-Caching verwenden oder man kann eine GET Request senden.

Aufgabe 1.6: HTML-Literatur lesen und Fragen beantworten

Arbeiten sie eine HTML-Tutorial durch (W3Schools oder selfhtml) und beantworten Sie dazu folgende Fragen:

  1. Was ist HTML?
  2. Wie kann man eine geschachtelte geordnete Liste der Tiefe 3 erzeugen?
  3. Wie ist eine HTML-Tabelle aufgebaut?
  4. Welche Konventionen sollte man bei Pfaden und Dateinamen beachten?
  5. Wie baut man in HTML ein Menü?
  6. Welche Attribute sollte man bei Bildern wie verwenden?

Meine Lösung:

  1. Eine Auszeichnungssprache
  2. nested list mit geschachtelten <ul> tags
  3. <table>, zeilen haben den tag <tr>, die spalten <th>, zeilen werden mit <td> gefüllt
  4. kleinschreibung, keine leerzeichen, nur buchstaben, zahlen und binde/-unterstriche, pfade im rootverzeichnis beginnen mit "/", pfade im jetzigen verzeichnis mit "./"
  5. <menu>
  6. src, alt und width/heigth. src beschreibt den Dateipfad. alt ist ein Alternativnamen für das Bild, falls es nicht geladen wird. height und width sind die Höhe und Breite des Bildes in Pixel.
